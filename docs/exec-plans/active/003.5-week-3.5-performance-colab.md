## 1. Goal

Week 3 produced a defensible 5-fold CV result (best: exp 11 triple weighted,
pooled F1 = 0.6000, CI [0.5814, 0.6206]). Week 3.5 is a **short, high-leverage
insert** to squeeze measurable macro-F1 gains out of the existing architecture
**without changing any locked design decision (PLD-\*)**, and to **migrate the
compute path to Colab A100** so the remaining runs are fast enough to finish
before the deadline.

Two parallel objectives:

1. **Performance lift** — add the imbalance-aware and inference-time techniques
   identified in the literature scan (focal loss, TTA, leakage-free seed
   ensemble; optional recipe-v2 tweaks) as **additive ablations** on top of the
   Week 3 best config. Original Week 3 CE runs are preserved for comparison.
2. **Colab/A100 migration** — make the training pipeline run reproducibly on a
   Colab Pro+ A100, with data staging, batch-size/LR scaling, and
   session-resilience, so the new runs cost minutes not hours.

By the end of this plan the project must have:

- A new **best config** (or a confirmed "no improvement" finding) with 5-fold CV
  + bootstrap CI, documented in `docs/results_progress.md` and traceable to
  `metrics.json`.
- A working, documented Colab A100 path (`env/requirements-colab.txt` filled,
  bootstrap notebook/script, verified A100 smoke test).
- `uv run pytest tests/` passing throughout.
- The best model **frozen** so Week 4 (confusion matrix, per-class table,
  Grad-CAM++) builds on the final checkpoint, not a stale one.

This does **not** change the Week 3 hard checkpoint result; it strengthens it.

## 2. Constraints (read before editing)

- **No PLD-\* change.** MLP-only classifier (PLD-06), torchvision-only backbones
  (PLD-15), 512 projection (PLD-04), concat/weighted mandatory (PLD-07), GMU as
  advanced (PLD-08) all stay locked. Everything in this plan fits inside them.
- **Focal loss is explicitly sanctioned, not a PLD change.** `project_plan.md §1`
  ("focal loss ... meaningful") and `§9` risk register
  ("focal loss ablation") both anticipate it. Reference: Lin et al. 2017
  (INDEX #24). It is added as an **ablation** — Week 3 CE runs are not deleted.
- **GPU training stays with the user.** Per standing instruction, any command
  that trains a model is **handed to the user to run** (now on Colab A100), never
  executed by the assistant. CPU-only post-processing (aggregate, CI, plots,
  TTA on cached logits) may be run by the assistant.
- **A100 only for Colab.** Use Colab only when an A100 is confirmed; reject T4
  (see memory: `project_compute_strategy.md`). RTX 5080 laptop remains a valid
  fallback.
- **Leakage discipline.** A naive "average all 5 fold models over the full
  dataset" **leaks** (each sample was in 4 folds' training sets). Only
  leakage-free ensembling is allowed (TTA per model; seed ensemble within a
  fold). Documented in Track B.
- **Honest reporting.** If a technique does not beat the Week 3 best at the
  95% CI level, that is the finding — report it, do not retune on the test set.
- **No BN-stat updates during fine-tune** unless an explicit ablation (AGENTS.md).
- **Plot after every experiment step** (memory: `feedback_visualization.md`).

## 3. Scope

### Track A — Focal Loss (imbalance-aware loss ablation) — PRIORITY 1
- Implement `FocalLoss` in `src/training/losses.py` (γ-focusing, optional
  per-class α from train-fold frequencies). Verbatim equation from Lin et al.
  2017 §3 in the docstring (AGENTS.md requirement for implemented equations).
- Extend `build_loss` to support `type: "focal"` and `type: "focal_balanced"`.
- Add `configs/training/finetune_wide_focal.yaml` (clone of `finetune_wide.yaml`,
  loss block swapped, everything else identical for a clean ablation).
- Add `tests/test_focal_loss.py`.
- Run the **Week 3 best config (exp 11, triple weighted)** with focal loss across
  5 folds on A100 → new experiment id `16_triple_weighted_finetune_focal_official`.

### Track B — Inference-time gains: TTA + leakage-free seed ensemble — PRIORITY 2
- Implement `scripts/evaluate.py` (currently a scaffold) to:
  - Load a trained run's checkpoint + its fold's test manifest.
  - **TTA:** average softmax over a fixed deterministic set of test-time
    transforms (identity, h-flip, and 1–2 light scales). No leakage; per-model.
  - Recompute macro-F1 / accuracy and write `metrics_tta.json` next to the run.
- **Seed ensemble (leakage-free):** for the chosen best config, train M=2–3 seeds
  **per fold** on A100 (same train split, different seed), average their softmax
  on that fold's held-out test set, then pool. This preserves the official
  5-fold protocol with zero leakage. (Naive cross-fold averaging is explicitly
  rejected — see Constraints.)
- Add `tests/test_tta.py` (shape + invariance: identity transform reproduces
  base logits; averaged probs still sum to 1).

### Track C — Recipe-v2 tweaks (optional, bundled) — PRIORITY 4
Only if Tracks A/B land with time to spare. Bundle into a **single** config to
avoid combinatorial blow-up:
- `configs/training/finetune_wide_v2.yaml`: patience 8→12, RandAugment M 9→12,
  add Mixup alongside CutMix, MLP hidden [256]→[512, 256].
- One 5-fold CV run on the best config with this bundle; keep only if it beats
  the Track A/B best at the CI level.

### Track D — Colab / A100 migration — PRIORITY 3 (enables A & B speed)
- Fill `env/requirements-colab.txt` from the working local stack (pin torch +
  torchvision CUDA wheels compatible with Colab's driver; the cu132 local pin
  likely needs a Colab-appropriate CUDA build — verify, don't assume).
- Add `notebooks/colab_bootstrap.ipynb` (or `scripts/colab_setup.sh`):
  clone repo → install → stage dataset + split manifests to local `/content`
  (not Drive-mounted reads) → verify `torch.cuda.get_device_name()` contains
  "A100" (assert + abort if T4) → run a 1-epoch smoke test.
- Add `configs/training/finetune_wide_a100.yaml`: batch_size 64→128, LR scaled
  (`head_lr` 1e-3→2e-3, `backbone_lr` 1e-4→2e-4), `cudnn_benchmark: true` for
  the exploration mode (PLD-18 allows config-controlled determinism).
- Confirm `run_cv.py` fold isolation survives a session drop (completed folds
  already saved; resume with remaining `--folds`).

### Track E — Documentation & decisions
- `docs/decisions.md`: new entries D-06 (focal loss added as ablation),
  D-07 (leakage-free ensembling policy — naive cross-fold averaging forbidden),
  D-08 (A100 batch/LR scaling for exploration mode).
- `docs/results_progress.md`: Week 3.5 subsection with the new ablation table +
  CI, and an explicit "new best vs Week 3 best" verdict.
- `docs/experiment_log.md`: dated entries per step group.
- `docs/known_issues.md`: any Colab-specific issue (driver/CUDA mismatch, etc.).

## 4. Out of Scope

- Any PLD-\* change (non-MLP classifier, timm backbones, projection dim change).
- Modern backbone swap (EfficientNetV2 / ConvNeXt) — that **requires a PLD-15
  relaxation and instructor sign-off**; deferred to a separate decision, not this
  plan. (Captured as an open question in §7.)
- SE/CBAM attention in fusion — Tier 3 idea; deferred unless A–D finish early.
- AFF / LMF fusion (stretch, PLD-09).
- Grad-CAM++, per-class table, confusion matrix → **Week 4** (this plan only
  ensures the best checkpoint is frozen for them).
- DeepWeeds, cross-protocol reproduction, UMAP, McNemar.
- Merging or dropping rare classes (forbidden — anti-hallucination §4.3).

## 6. Step-by-Step Implementation Plan

> One step at a time. After each step: run tests, update docs, **stop and wait
> for approval** before the next step.

### ⬜ Step 1 — Colab A100 migration first (unblocks fast training)
Do this before the training-heavy tracks so every later run is cheap.
- Fill `env/requirements-colab.txt`; create the bootstrap (clone, install, stage
  data to `/content`, A100 assert, 1-epoch smoke test).
- Add `configs/training/finetune_wide_a100.yaml` (batch 128, LR ×2, cudnn_benchmark).
- **User runs** the smoke test on Colab; paste back: device name (must contain
  A100), per-epoch time, no-NaN confirmation.
- Acceptance: smoke test completes on A100; per-epoch time recorded; resume-after-
  drop verified by killing and re-running one fold.

### ⬜ Step 2 — Focal loss implementation + tests
- Implement `FocalLoss` (Lin 2017 eq. verbatim in docstring) + `build_loss`
  extension + `finetune_wide_focal.yaml`.
- `tests/test_focal_loss.py`: reduces to weighted CE at γ=0; down-weights
  easy examples at γ>0; α-vector shape matches class count; no NaN at p→0
  (numerical stability); gradient flows.
- Run `uv run pytest tests/ -v` (CPU, assistant can run) — no regressions.
- Acceptance: all tests pass; equation in docstring matches paper.

### ⬜ Step 3 — Focal 5-fold CV on best config (A100)
- Add exp 16; **user runs** `run_cv.py` for `16_..._focal` folds 0–4 on A100.
- Assistant runs `aggregate_cv.py` + `compute_ci.py` + `generate_report_tables.py`
  + plots afterward (CPU).
- Acceptance: 5 folds present, no NaN; CV mean±std + CI computed; verdict written
  (focal vs Week 3 best at CI level).

### ⬜ Step 4 — TTA inference (no training)
- Implement `scripts/evaluate.py` TTA path + `tests/test_tta.py`.
- Assistant runs TTA over existing best-config checkpoints (CPU/GPU light) →
  `metrics_tta.json`; recompute pooled CI.
- Acceptance: TTA macro-F1 reported vs non-TTA for the best config; tests pass.

### ⬜ Step 5 — Leakage-free seed ensemble (A100, optional-but-recommended)
- **User runs** M=2–3 seeds per fold for the best config; assistant pools per
  fold (held-out only) → ensemble CI.
- Acceptance: ensemble result documented; leakage-free method note in decisions.md.

### ⬜ Step 6 — Recipe-v2 bundle (optional, only if time remains)
- `finetune_wide_v2.yaml`; one 5-fold CV on best config; keep only if it wins at
  the CI level. Otherwise document as "no improvement."

### ⬜ Step 7 — Freeze best model + close-out
- Decide the **final best config** for Week 4 (highest CI-lower-bound, honest).
- Update `results_progress.md` Week 3.5 section (ablation table + CI + verdict).
- Append Week 3.5 close-out to `experiment_log.md`; update `known_issues.md`.
- `uv run pytest tests/ -v` green.
- Move this plan to `docs/exec-plans/completed/`.

## 7. Risks / Open Questions

| Risk / Question | Likelihood | Mitigation |
|---|---|---|
| Focal loss yields no CI-level gain over CE | Medium | Honest finding; CE best remains; cost was one config × 5 folds |
| Naive fold ensemble leakage slips in | Low | Explicitly forbidden in D-07; only TTA + per-fold seed ensemble allowed |
| Colab cu132 wheel mismatch with A100 driver | Medium | Verify Colab CUDA build empirically in Step 1; pin a Colab-appropriate wheel, don't copy the local cu132 pin blindly |
| Colab assigns T4 not A100 | Medium | Bootstrap asserts device name contains "A100" and aborts; fall back to RTX 5080 |
| Session drop mid-CV on Colab | Medium | `run_cv.py` per-fold isolation; resume with remaining `--folds`; stage data to /content (not Drive) |
| LR ×2 at batch 128 destabilizes training | Low-Med | Warmup already 5 epochs; if val_f1 collapses, fall back to batch 64 / LR ×1 |
| Time runs out before Week 4 | Medium | Strict priority order: D→A→B are the must-haves; C and E5 are optional |
| **Open question:** modern backbone (EffNetV2/ConvNeXt) would add +2–4 F1 but needs PLD-15 relaxation | — | **Ask instructor before any work**; out of scope for this plan |
